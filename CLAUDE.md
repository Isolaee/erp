# Coding ERP — Project Context

## What This Is
An engineering team coordination tool. Tracks what each programmer is doing, manages
hierarchical task lists (org/team/personal), and uses Anthropic Claude AI to refine
task descriptions and distribute work. All infrastructure runs in Docker.

## Tech Stack
- **Backend**: Node.js + Express + TypeScript (`backend/src/`), Prisma ORM, PostgreSQL 16
- **Frontend**: React + TypeScript + Vite + TailwindCSS (`frontend/src/`)
- **LLM**: Anthropic Claude API with tool use (`@anthropic-ai/sdk`)
- **Realtime**: Server-Sent Events (SSE) — one-way push from backend to clients
- **Cache/Sessions**: Redis 7 (refresh token blacklist, GitHub API cache 5-min TTL)
- **Auth**: JWT — access token (15min, in-memory on client), refresh token (7d, httpOnly cookie)

## Development

### Start everything
```bash
docker compose up --build
```
Services: `postgres:5432`, `redis:6379`, `backend:3001`, `frontend:5173`

### Backend only (requires local postgres + redis or running docker deps)
```bash
cd backend && npm run dev
```

### Database
```bash
cd backend && npx prisma migrate dev --name <name>   # create migration

```

```bash

```

## Key Architecture Decisions

### Access Control
`backend/src/services/accessControl.ts` — `canUserAccessList()` and `canUserAccessTask()`
are the single source of truth for visibility rules. Every route that returns list/task
data MUST call these helpers. Do not inline access checks in route handlers.

**Visibility rules:**
- `ORGANIZATION` scope + `ORGANIZATION` visibility → all org members
- `TEAM` scope → team members only (unless visibility overridden to ORGANIZATION)
- `PERSONAL` scope + `PRIVATE` visibility → owner only, BUT ADMIN always sees all, TEAM_LEAD sees their team members' personal lists
- Creator can always override visibility at list level

### LLM Tool Use (Single-Turn Loop)
`backend/src/services/aiService.ts` — one round of Claude tool calls per `/api/ai/refine`
request. Claude sees context → calls tools → gets results → writes summary. Intentionally
not recursive to keep the POC predictable. Tool definitions live in `aiService.ts`.

**Available tools:** `create_task`, `update_task`, `move_task`, `assign_task`, `delete_task`

### Task Ordering
`Task.order` is a `Float`. Insert between two tasks: `newOrder = (prev + next) / 2`.
No complex rank strings. Works fine at POC scale.

### SSE Registry
`backend/src/routes/events.ts` holds an in-memory `Map<userId, Response>`. Sufficient for
single-instance POC. Upgrade path: Redis pub/sub per channel for multi-instance.

### No Email for POC
Invite links are returned in the API response — admin copies and shares manually.
No SMTP service. Add nodemailer + Resend in a later iteration.

### Role Hierarchy
Global: `ADMIN > TEAM_LEAD > MEMBER`
`TeamMember.role` is independent from `User.role` (a global MEMBER can be a TEAM_LEAD within a specific team).

## Project Structure
```
backend/
  prisma/
    schema.prisma        all models — source of truth for DB shape
    migrations/          auto-generated by prisma migrate
    seed.ts              creates admin user + demo team
  src/
    index.ts             Express app entry, mounts all routers
    config.ts            env vars validated with Zod (fail fast on missing vars)
    middleware/
      auth.ts            verifyAccessToken — attaches req.user
      roles.ts           requireRole(UserRole) — 403 if insufficient
      errorHandler.ts    global error handler
    routes/
      auth.ts            login, refresh, logout, me, register
      users.ts           user CRUD
      teams.ts           team CRUD + members + repo follows
      invites.ts         create, list, revoke, preview, accept
      lists.ts           task list CRUD
      tasks.ts           task CRUD + move + assign + assignment respond
      github.ts          proxy to GitHub REST API (cached)
      ai.ts              POST /refine → SSE stream with tool execution
      events.ts          GET /events → SSE connection per user
    services/
      authService.ts     JWT sign/verify, bcrypt, refresh token DB ops
      aiService.ts       Claude API call, tool definitions, streaming loop
      sseService.ts      emit(userId, type, payload) — writes to SSE registry
      githubService.ts   octokit wrapper with Redis TTL cache
      accessControl.ts   canUserAccessList, canUserAccessTask
    lib/
      prisma.ts          singleton PrismaClient (import from here, never new PrismaClient())
      redis.ts           singleton ioredis client

frontend/
  src/
    main.tsx
    App.tsx              React Router setup, AuthProvider, QueryClientProvider, SSEProvider
    components/
      layout/            AppShell, Sidebar, TopBar
      auth/              LoginForm, RegisterForm, ProtectedRoute, RoleGuard
      users/             UserAvatar, UserCard, UserSelect
      teams/             TeamCard, TeamMemberRow, RepoFollowCard, AddRepoModal
      lists/             ListCard, CreateListModal, ListScopeTag
      tasks/             TaskCard, TaskStatusBadge, CreateTaskModal, TaskAssignmentPanel, SubtaskList
      ai/                RefinementPanel, ToolCallBadge
      github/            RepoIssueList, RepoPRList, RepoCommitList
      shared/            Modal, Button, Input, Badge, Spinner, EmptyState
    pages/               one file per route
    hooks/               useAuth, useLists, useTasks, useTeams, useSSE
    context/             AuthContext, SSEContext
    lib/
      api.ts             axios instance — auto-refresh interceptor, all requests go through here
      queryClient.ts     React Query client config
    types/
      api.ts             TypeScript types mirroring Prisma models (kept in sync manually)
```
